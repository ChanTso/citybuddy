# CB-013 — RocketMQ Broker and Proxy foundation

## Referenced contracts

- [Executable contract and version truth](../CONTRACTS.md#contract-executable-truth)
- [RocketMQ runtime preflight](../CONTRACTS.md#contract-preflight-rocketmq-runtime)

## Goal

Provide a pinned, health-gated RocketMQ 5 Broker and Proxy path and prove a normal-message round trip through the selected client path.

## In scope

- Extend the local Compose topology with pinned RocketMQ NameServer, Broker, and Proxy components.
- Expose and verify the RocketMQ 5 Proxy endpoint used by later clients.
- Add readiness checks that verify Broker/Proxy service behavior rather than only container state or an open port.
- Add a disposable topic and minimal producer/consumer round trip through the selected runtime client path.
- Record exact Broker, Proxy, and probe-client versions and configuration.

## Acceptance criteria

1. `make up` returns only after NameServer, Broker, and Proxy are ready for the selected client path.
2. The advertised endpoint is the RocketMQ 5 Proxy endpoint; the probe does not bypass it with a different protocol or direct Broker path.
3. A uniquely identified normal message is produced and consumed once by the probe, with bounded wait and explicit cleanup.
4. A controlled Broker or Proxy readiness failure makes startup non-zero and visible.

## Rejection paths

- Reject RocketMQ without a reachable 5.x Proxy path or a probe that bypasses the selected runtime path.
- Reject process-only, port-only, or sleep-based readiness when a client-level check is available.
- Reject claims about transaction-message, delay-message, acknowledgement, redelivery, or long-processing semantics based on the normal-message probe.
- Reject production topics, business payloads, consumers, or event schemas in this slice.

## Out of scope

- Python consumer viability and acknowledgement behavior owned by `CB-085`;
- transaction and delay fault semantics owned by `CB-060`;
- production topics, Outbox publishing, and business consumers;
- aggregate runtime closure.

## Required evidence

- Pinned Broker, Proxy, NameServer, and probe-client provenance;
- endpoint/configuration and meaningful readiness output;
- normal-message producer/consumer round-trip output with identifiers;
- controlled Broker or Proxy failure with non-zero startup;
- real Branch, PR, Commits, Tests, and Notes only at completion.
