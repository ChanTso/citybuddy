## Goal

Establish an executable baseline proving that the current repository has no evaluation-reachable asynchronous producer/consumer path, close every current path as production-only or unreachable with real component evidence, and prevent reserved sandbox metadata from upgrading a production message.

## In scope

- Inventory exactly the six current asynchronous or apparent-asynchronous entry families: seckill transaction orders, seckill unpaid timeouts, product publication, standard-order/refund Outbox rows, the disposable knowledge-indexer spike, and agent-service.
- Classify each family as production-only, unreachable from evaluation, not a runtime path, or having no asynchronous runtime. Record zero evaluation-reachable paths.
- Exercise real eval_direct_user JWTs against the real seckill reservation, standard-order, and refund controllers. Each production-only controller must reject before its coordinator or service is invoked.
- Keep the current production message schemas free of sandbox fields. The seckill transaction, seckill timeout, and catalog consumers reject the reserved citybuddy-eval-sandbox-id envelope property before invoking their handlers.
- Use real RocketMQ Broker delivery to prove the three current production consumers reject that reserved property without a business handler, order/ledger, cancellation, or catalog handler mutation.
- Prove that only product-publication Outbox rows have a current runtime publisher, that the knowledge-indexer messaging implementation remains a disposable spike with no sandbox field, and that agent-service has no asynchronous runtime producer or consumer.
- Add no liveness API, liveness lookup, guard, drop/archive disposition, sandbox column, migration, runtime grant, retry behavior, callback behavior, or evaluator implementation.

## Acceptance criteria

1. One executable inventory contains exactly the six named path families and reports evaluationReachablePathCount = 0; every row has a concrete classification and evidence.
2. A cryptographically valid eval_direct_user JWT with a matching sandbox header is sent through real MVC request handling to the real seckill reservation, standard-order, and refund controllers. Every request returns the fixed production-only authentication failure, and the downstream coordinator/service has no interaction.
3. SeckillTransactionMessage, SeckillTimeoutMessage, and the catalog publication event have no sandbox field. The current producers do not add the reserved sandbox envelope property.
4. The real RocketMQ catalog, seckill transaction, and seckill timeout consumers reject citybuddy-eval-sandbox-id before invoking the corresponding handler or changing order, ledger, cancellation, or catalog state.
5. The catalog Outbox publisher selects only PRODUCT_PUBLICATION_CHANGED; standard-order and refund Outbox rows have no current runtime publisher and cannot be presented as delivered asynchronous work.
6. The CB-085 knowledge-indexer SpikeEvent has no sandbox field and its production worker has no messaging behavior. Agent-service contains no RocketMQ or durable asynchronous publisher/consumer runtime.
7. Production messages without the reserved property retain their existing schemas and behavior. CB-104 adds no sandbox-aware business mutation and does not enable evaluation access to production controllers.
8. The frozen asynchronous contract states that the first future slice introducing an evaluation-reachable asynchronous producer/consumer must implement the complete liveness guard and real Broker evidence in that same slice, using this inventory as the introduction baseline.
9. No runtime liveness API, database disposition, sandbox business column, cross-service database access, broad grant, mock-payment callback extension, ServiceEval, evaluator score/verdict, web, deployment, or later behavior is introduced.
10. Static inventory/schema checks, controller boundary tests, real Broker integration, and complete make ci pass.

## Transferred guard obligation

CB-104 does not claim or test sandbox liveness guard behavior because the executable inventory proves that no current evaluation request can legally create asynchronous work. Redelivery, completion races, restart recovery, liveness outage handling, and idempotent drop/archive evidence are mandatory acceptance criteria of the first future slice that makes any inventory row evaluation-reachable; they are not waived or satisfied by CB-104.

## Rejection paths

- Reject an inventory that omits one of the six families, infers reachability from a mock, or treats direct repository/coordinator calls, database fixture insertion, or hand-built sandbox messages as a legal evaluation producer.
- Reject switching any production controller to evaluation authorization or adding sandbox business mutations to seckill, order, inventory, ledger, quota, refund, or cache truth.
- Reject a sandbox field in a current production payload or acceptance of the reserved sandbox property by a production consumer.
- Reject claims that standard-order/refund Outbox rows are delivered when no runtime publisher selects them, that the CB-085 spike is a runtime worker, or that synchronous agent routes are asynchronous.
- Reject any liveness endpoint, guard, disposition/archive schema, migration, grant, retry, callback, or broker-success claim carried over from the discarded recovery draft.
- Reject production behavior changes, mock-only controller evidence, or consumer-property checks that never traverse the real Broker.

## Out of scope

- sandbox-isolated seckill activity, product, inventory, ledger, quota, order, payment, refund, or cache truth;
- sandbox liveness guards and drop/archive behavior before a real evaluation-reachable asynchronous carrier exists;
- sandbox-bound mock-payment callbacks owned by CB-105;
- ServiceEval, evaluator scoring, retrieval/reranking, actions/receipts, memory, handoff, web, deployment, provider access, and performance evidence.

## Required evidence

- executable six-row inventory and static source/schema assertions;
- real signed evaluation JWT requests through the production-only controllers with downstream non-invocation assertions;
- real Broker reserved-property rejection for catalog, seckill transaction, and seckill timeout consumers, plus no-mutation checks;
- exact Outbox selection, disposable-spike, no-agent-async-runtime, and no-guard-artifact assertions;
- passing real make ci and exact additional integration commands recorded in the pull request;
- real Branch, PR, Commits, Tests, and Notes only at completion.
