## Goal

Establish one reproducible Elasticsearch 8 and IK compatibility pair and prove the component semantics required by later retrieval slices without creating a business index.

## In scope

- Add a pinned Elasticsearch 8 runtime with a version-matched, reproducibly installed IK plugin.
- Record exact image/plugin provenance in executable configuration.
- Add meaningful cluster and analyzer readiness checks.
- Exercise a disposable mixed-language analysis request, `dense_vector` mapping, minimal kNN query, and atomic test-alias switch.
- Extend local startup, shutdown, and integration checks with these disposable probes.

## Acceptance criteria

1. Elasticsearch becomes healthy only after the selected IK analyzer is loaded and usable.
2. A representative mixed-language query produces analyzer output through the pinned IK plugin.
3. A disposable index accepts a `dense_vector` mapping and returns the expected document from a minimal kNN query.
4. One atomic aliases operation moves a disposable read alias from a validated old index to a validated new index.
5. Disposable probe data is cleaned without creating the later `knowledge_docs_vN` business projection.

## Rejection paths

- Reject startup without a verified IK analyzer or a silent fallback to another analyzer.
- Reject unpinned Elasticsearch/plugin compatibility, manual host installation, or plugin provenance that cannot be reproduced.
- Reject a vector test that only creates a mapping without querying it, or an alias test that mutates an unvalidated target.
- If no reproducible matching IK artifact or build path exists, mark `CB-012` `BLOCKED`; do not change analysis behavior silently.

## Out of scope

- Production knowledge indexes, embeddings, RAG, RRF, reranking, publication, tombstones, or rebuild automation;
- private or transactional data in Elasticsearch;
- RocketMQ and aggregate runtime closure.

## Required evidence

- Exact Elasticsearch image and IK artifact/build provenance;
- health and plugin inventory output;
- analyzer, dense-vector, kNN, cleanup, and atomic-alias output;
- a concise result for `SP-002` and any blocking evidence;
- real Branch, PR, Commits, Tests, and Notes only at completion.
